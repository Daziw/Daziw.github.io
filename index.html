2026-04-16
根据巡查统计需求，对采购系统单据数据进行每日数量统计，优化并使用SQL语句实现按日统计单据数量功能，SQL语句如下：
SELECT DATE(VOU_DATE) AS 日期, COUNT(AGENCY_CODE) AS 单据数量 FROM gl_vou_head WHERE VOU_DATE >= '2026-01-01' AND VOU_DATE < '2026-02-01' GROUP BY DATE(VOU_DATE) ORDER BY DATE(VOU_DATE);
本 SQL 语句用于按日统计单据数量，通过 SELECT 指定查询日期与单据数量字段，使用 AS 别名显示具体日期，FROM 指定数据源表，WHERE 筛选 1 月份时间范围，GROUP BY 按天分组实现每日统计，ORDER BY 按日期排序展示结果，最终输出每日单据数量及对应日期，使数据清晰直观，解决了原查询无法显示日期、无数据日期自动隐藏的问题，完成 1-3 月单据数据核对与报表整理工作。

学习并实践基于 Docker 容器化部署 mall-web 管理系统，编写 Dockerfile 配置文件，使用阿里云镜像库的 OpenJDK 8 作为基础镜像，将本地 Tomcat 7 和 WAR 包复制到容器中；使用 docker-compose 编排 Redis 和 mall-web 双服务，配置网络通信和数据持久化；执行构建部署命令完成容器化部署，应用成功运行在 8080 端口。掌握了 Docker 常用管理命令，理解了容器化部署在环境一致性、快速部署方面的优势。总结 Docker 常用命令：
# 启动所有服务（后台运行） docker-compose up -d # 重新构建并启动（代码变更后） docker-compose up -d --build # 停止服务 docker-compose stop # 重启服务 docker-compose restart # 停止并删除容器 docker-compose down # 查看运行中的容器 docker-compose ps # 查看实时日志 docker-compose logs -f mall-web # 进入容器内部 docker exec -it fjgp-mall-web bash # 查看容器资源使用 docker stats
通过以上命令可以完成容器的全生命周期管理：up 命令用于启动服务，down 命令用于清理容器，logs 命令用于排查问题，exec 命令用于进入容器调试。相比传统部署方式，Docker 容器化部署避免了环境配置差异导致的各种问题，一键即可启动包含 Redis 缓存和 Web 应用的完整系统，大大提升了部署效率和运维便利性。
